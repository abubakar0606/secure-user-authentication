Two-Factor Authentication was implemented using **Google Authenticator**.
After entering a username and password, users must provide a time-based one-time
password (OTP) generated by the authenticator app. This adds an extra layer of
security and prevents unauthorized access even if passwords are compromised.


 2Ô∏è‚É£ **OAuth 2.0 Authentication**
OAuth 2.0 was used to enable secure and seamless user sign-in.
Users can authenticate using third-party providers (such as Google) without sharing
their passwords with the application. Instead, access tokens are used to grant
temporary and limited access.

This approach improves both security and user experience.


3Ô∏è‚É£ **AES-256 Encryption**
Sensitive user data such as passwords and authentication tokens were protected using
**AES-256 encryption**. AES-256 is a strong symmetric encryption algorithm widely used
in enterprise and cloud environments.

Encryption ensures that even if data is exposed, it remains unreadable to unauthorized users.
